# هيكل قاعدة البيانات — نظام شاشة المسجد الذكية

**الوثيقة الكاملة والمفصّلة:** [قاعدة البيانات — وثيقة كاملة](database-firestore-complete.md) (تشمل Firestore، adhan_dart، QR داخل الإعلان، وجميع الحقول).

## نظرة عامة

قاعدة البيانات مركزية على **Firebase Firestore**. أوقات الصلاة تُحسب في التطبيق باستخدام مكتبة **adhan_dart** من إحداثيات المسجد و**`calculation_method`** (ديناميكي — يتحكم فيه المسجد من لوحة التحكم). محتوى **رمز QR داخل كل إعلان** مخزَن في `qr_content` ويُولَّد في التطبيق.

---

## الحسابات وربط المسجد (تسجيل الدخول)

لتمكين "نفس الحساب" على الجوال والشاشات، يُربط المستخدم بالمسجد في Firestore:

### المستخدمون (Firebase Authentication)

- **Firebase Authentication** يحفظ هوية المستخدم (uid، بريد، إلخ).
- لا تُخزَن كلمات المرور في Firestore؛ الاعتماد على Auth فقط.

### ربط المستخدم بالمسجد

يجب أن يعرف التطبيق: "هذا الحساب يتحكم بأي مسجد؟" لإظهار الإعدادات أو عرض بيانات المسجد الصحيح.

**أحد النماذج المقترحة:**

| المجموعة/المستند | الوصف |
|-------------------|--------|
| `users/{uid}` | مستند للمستخدم (اختياري: اسم، صورة، إعدادات محلية). |
| `users/{uid}/mosques` (مجموعة فرعية) أو `user_mosques` (مجموعة) | ربط المستخدم بالمساجد: كل عنصر يحتوي `mosque_id` وربما `role` (مثلاً: admin، editor). |

**بديل أبسط:** مستند المسجد يحتوي حقل `owner_uid` أو `allowed_uids` (قائمة معرّفات مستخدمين مسموح لهم القراءة/الكتابة). عند تسجيل الدخول نتحقق من أن uid موجود في المسجد المرتبط.

- **النتيجة:** عند فتح التطبيق (جوال أو شاشة) بعد تسجيل الدخول، نعرف مسجد المستخدم ونقرأ/نكتب مستند هذا المسجد.

## مستند المسجد (Mosque Document)

### الحقول الأساسية

| الحقل | النوع | الوصف |
|-------|------|--------|
| `name` | string | اسم المسجد |
| `logo_url` | string | رابط شعار المسجد |
| `city` | string | المدينة |
| `latitude` | number | خط العرض |
| `longitude` | number | خط الطول |
| `calculation_method` | string | طريقة حساب أوقات الصلاة — **ديناميكي** من لوحة التحكم (مثل: `umm_al_qura`, `muslim_world_league`, `egyptian`…). التطبيق يمررها لمكتبة adhan_dart. |
| `updated_at` | string (ISO 8601) | آخر تحديث للبيانات |
| `last_seen` | string (ISO 8601) | آخر نشاط/اتصال |

### إعدادات الإقامة — `iqama_offsets`

وقت الإقامة = وقت الأذان + عدد الدقائق (offset).

| الحقل | النوع | الوصف |
|-------|------|--------|
| `iqama_offsets.fajr` | number | دقائق بعد أذان الفجر |
| `iqama_offsets.dhuhr` | number | دقائق بعد أذان الظهر |
| `iqama_offsets.asr` | number | دقائق بعد أذان العصر |
| `iqama_offsets.maghrib` | number | دقائق بعد أذان المغرب |
| `iqama_offsets.isha` | number | دقائق بعد أذان العشاء |

**لماذا offsets وليس أوقاتاً ثابتة؟**  
لا نخزن أوقاتاً ثابتة؛ النظام ديناميكي ويحسب وقت الإقامة دائماً من وقت الأذان + الـ offset، مما يسهل التعديل ويتكيف مع تغيّر أوقات الصلاة.

### إعدادات التصميم — `design_settings`

| الحقل | النوع | الوصف |
|-------|------|--------|
| `background_type` | string | نوع الخلفية: `image` \| `color` \| `gradient` |
| `background_value` | string | رابط صورة أو لون أو قيمة التدرج |
| `primary_color` | string | اللون الرئيسي (مثل: `#1E3A8A`) |
| `secondary_color` | string | اللون الثانوي |
| `font_family` | string | نوع الخط (مثل: `Amiri`) |
| `font_size` | number | حجم الخط |
| `updated_at` | string (ISO 8601) | آخر تحديث للإعدادات |

### الأحاديث — `hadiths` (مصفوفة)

| الحقل | النوع | الوصف |
|-------|------|--------|
| `narrator` | string | الراوي |
| `text` | string | نص الحديث |
| `source` | string | المصدر (مثل: صحيح البخاري) |
| `display_order` | number (اختياري) | ترتيب العرض |
| `duration_seconds` | number (اختياري) | مدة عرض الحديث بالثواني |
| `updated_at` | string (ISO 8601) | آخر تحديث |

### إعلانات المسجد — `mosque_ads` (مصفوفة)

كل إعلان يعرض على الشاشة ويمكن أن يحتوي **رمز QR**؛ المحتوى داخل الإعلان في `qr_content` والتطبيق يولد الـ QR منه (مثل qr_flutter).

| الحقل | النوع | الوصف |
|-------|------|--------|
| `title` | string | عنوان الإعلان |
| `start_date` | string (YYYY-MM-DD) | تاريخ البداية |
| `end_date` | string (YYYY-MM-DD) | تاريخ النهاية |
| `qr_content` | string | **محتوى رمز QR** — رابط أو نص يُولَّد منه رمز الـ QR داخل الإعلان |
| `image_url` | string (اختياري) | صورة مصاحبة للإعلان |
| `updated_at` | string (ISO 8601) | آخر تحديث |

---

## منطق الحساب

- **أوقات الصلاة:** تُحسب في التطبيق بمكتبة **adhan_dart** من `latitude`، `longitude`، و`calculation_method` (ديناميكي من Firebase). لا تُخزَّن أوقات يومية في Firebase.
- **وقت الإقامة للصلاة X** = وقت أذان الصلاة X + `iqama_offsets[X]` (بالدقائق).
- **العد التنازلي:** الصلاة القادمة و`timeUntilNextPrayer()` من المكتبة؛ ثم العد حتى الإقامة.
