# الخطة الشاملة — نظام شاشة المسجد الذكية

نظرة عامة على مراحل المشروع. التكامل مع **Firebase** (Firestore + Realtime + Authentication).  
**المبدأ:** تطبيق واحد ذكي يمييز البيئة (جوال / شاشة) من السبلاش، وتسجيل دخول للجميع؛ الجوال يغيّر الإعدادات ويرفعها، والشاشات المسجلة على نفس الحساب تسحب التغييرات وتطبّقها.

---

## المعمارية الأساسية (تطبيق واحد ذكي)

| العنصر | الوصف |
|--------|--------|
| **تطبيق واحد** | نفس التطبيق يعمل على الجوال وعلى الشاشة (جهاز العرض). |
| **تمييز البيئة** | عند التشغيل (من السبلاش) يُحدَّد هل الجهاز جوال أم شاشة (حجم شاشة و/أو اختيار مخزَن محلياً). |
| **السبلاش أساس التوجيه** | سبلايش → تسجيل دخول (إن لزم) → تمييز البيئة → اختيار المسجد → إما واجهة الإعدادات (جوال) أو واجهة العرض (شاشة). |
| **واجهة الإعدادات (جوال)** | تعديل كل إعدادات المسجد (البيانات، التصميم، الأحاديث، الإعلانات، أوقات الإقامة) ورفعها إلى Firebase. |
| **واجهة العرض (شاشة)** | سحب بيانات المسجد من Firebase (مع التحديث الفوري) وعرضها: عد تنازلي، إقامة، أحاديث، إعلانات. |
| **تسجيل الدخول** | مطلوب على الجوال والشاشة؛ الحساب يربط المستخدم بمسجد. الأجهزة المسجلة على نفس الحساب = نفس المسجد. |
| **سحب التغييرات** | الشاشات تشترك في مستند المسجد (Realtime)؛ أي تغيير يرفعه الجوال يُسحَب فوراً وتُطبَّق على الشاشة. |

التفاصيل الكاملة: [معمارية التطبيق الذكي](architecture-smart-app.md).

## المرحلة الأولى — تصميم وبناء البنية الأساسية

**الحالة:** مكتملة (تصميم وتخطيط).

**الأهداف:**
- تصميم قاعدة بيانات مرنة (مسجد، إقامة، تصميم، أحاديث، إعلانات، ربط مستخدم–مسجد).
- تحديد معمارية التطبيق الواحد الذكي وتمييز البيئة.
- تحديد آلية حساب الصلاة والإقامة (وقت الإقامة = وقت الأذان + offset).
- تجهيز نموذج البيانات (يشمل `iqama_offsets`، `design_settings`، `hadiths`، `mosque_ads`).

**المخرجات:**
- مخطط قاعدة البيانات (`database-schema.md`، `schema-mosque-example.json`).
- معمارية التطبيق الذكي (`architecture-smart-app.md`).
- تقرير المرحلة الأولى (`01-phase-1-report.md`).

## المرحلة الثانية — السبلاش، تسجيل الدخول، وتمييز البيئة

**الأهداف:**
- بناء تدفق السبلاش: تحميل → التحقق من تسجيل الدخول → تمييز البيئة (جوال / شاشة) → تحديد المسجد → توجيه إلى الإعدادات أو العرض.
- تنفيذ تسجيل الدخول (Firebase Auth) للجوال والشاشة.
- ربط الحساب بالمسجد في Firestore (وفق هيكل الحسابات في `database-schema.md`).
- آلية تمييز البيئة (حجم شاشة و/أو اختيار مخزَن محلياً).

**التكامل مع Firebase:**
- Firebase Authentication لتسجيل الدخول.
- Firestore لقراءة ربط المستخدم بالمسجد وتحديد مستند المسجد.

**المخرجات:**
- سبلايش يعمل وفق المعمارية الذكية.
- تسجيل دخول موحّد وتوجيه صحيح لواجهة الإعدادات أو العرض.

## المرحلة الثالثة — واجهة الإعدادات ورفع البيانات إلى Firebase

**الأهداف:**
- بناء واجهة الإعدادات (تعرض فقط في وضع الجوال بعد السبلاش): بيانات المسجد، التصميم، الأحاديث، الإعلانات، أوقات الإقامة (`iqama_offsets`).
- رفع كل التعديلات إلى Firestore (مستند المسجد المرتبط بالحساب).
- تحديث `updated_at` و`last_seen` عند الحفظ.

**التكامل مع Firebase:**
- الكتابة والتحديث على مستند المسجد من التطبيق (وضع الجوال).

**المخرجات:**
- واجهة إعدادات كاملة ورفع موثوق إلى Firebase.

---

## المرحلة الرابعة — واجهة العرض وسحب التغييرات الفورية

**الأهداف:**
- بناء واجهة العرض (تعرض فقط في وضع الشاشة بعد السبلاش): عد تنازلي، إقامة، أحاديث، إعلانات، وفق إعدادات التصميم.
- الاشتراك في مستند المسجد عبر Firestore Realtime (Snapshot listener).
- سحب التغييرات فوراً وتطبيقها على الواجهة (بدون إعادة تشغيل).

**التكامل مع Firebase:**
- قراءة مستند المسجد + استماع فوري للتحديثات.

**المخرجات:**
- شاشة عرض تتحدث تلقائياً مع أي تغيير من أي جهاز على نفس الحساب.

---

## المرحلة الخامسة — تجربة تشغيل حقيقية والتحسين

**الأهداف:**
- تجربة تشغيل في مسجد (أو بيئة مشابهة): جوال واحد + شاشة واحدة أو أكثر، نفس الحساب.
- التحقق من: رفع الإعدادات من الجوال، ظهور التغييرات فوراً على الشاشة، دقة العد التنازلي ووقت الإقامة.
- تحسين الأداء والاستقرار (كاش، انقطاع الشبكة، إعادة الاتصال).

**المخرجات:**
- تقرير تجربة تشغيل وتوثيق المرحلة الخامسة.

---

## المرحلة السادسة — الأمان والنشر والتوثيق

**الأهداف:**
- قواعد أمان Firestore: القراءة/الكتابة حسب المستخدم والمسجد (ربط الحساب بالمسجد).
- إعداد بيئة النشر (تطبيق واحد: جوال + شاشة).
- توثيق التشغيل والصيانة وإدارة الحسابات.

**المخرجات:**
- نظام جاهز للنشر ووثائق تشغيل.

---

## ملخص التكامل مع Firebase

| العنصر | الاستخدام |
|--------|-----------|
| **Firebase Authentication** | تسجيل الدخول على الجوال والشاشة؛ ربط المستخدم بالمسجد. |
| **Firestore** | مستند مسجد لكل مسجد (البيانات، `iqama_offsets`، `design_settings`، `hadiths`، `mosque_ads`)؛ مجموعات/مستندات لربط المستخدمين بالمساجد. |
| **Realtime updates** | الشاشات تشترك في مستند المسجد؛ أي تغيير من الجوال يصل فوراً وتُطبَّق التحديثات. |
